1. Многозадочность
	- в системе организовано работа так, что она может хранить в памяти одновременно несколько программ и данные.
При этом:
- можно переключаться между задачами
- защита данных и программ каждой задачи

2. Процесс
	- вся программа с ее описаниями

3. Поток
	- только код (типо сегмент кода)


FCFS
+ простота реализации
+ нет постоянного откладывания
+ малые ресурсы на формирование очереди
- нет быстрого освобождения ресурсов

SJN, SRT
+ быстрое освобождение ресурсов
- есть возможность бесконечного откладывания
- затраты на формирование очереди
- время указывает программист

RR
+ простота реализации
+ минимальные затраты на поддержку очереди
- выделение кванта времени одиноково. Если процессу надо меньше времени и он закончит работу, то процессор будет простаивать

Windows NT
+ квант времени от приоритета
+ важные вперед
- сложность перемещения

Unix
+ быстрота обслуживания (ввод/вывод - приоритет повышается)
- перемещение приоритетов

Linux
- откладывание
+ простота поддержки очереди
